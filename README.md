# RuEduCMS

CMS для создания сайтов образовательных учреждений России. Быстрая установка, встроенные модули, соответствие требованиям законодательства.

## Возможности

- **5-минутная установка** — распакуйте ZIP, откройте сайт, введите данные БД
- **Управление контентом** — страницы, новости, медиабиблиотека, меню
- **Встроенные модули** — сведения об ОО, педагоги, расписание, документы, галерея, контакты, формы
- **Система тем** — PHP-шаблоны + SCSS (как OpenCart)
- **Законодательство РФ** — раздел «Сведения об ОО», версия для слабовидящих, ФЗ-152, cookie-баннер
- **SEO** — ЧПУ, meta-теги, Open Graph, sitemap.xml, Schema.org

## Требования

- PHP 8.2+
- MySQL 5.7+ / MariaDB 10.3+
- Расширения: pdo_mysql, mbstring, json, gd
- Apache с mod_rewrite **или** nginx + PHP-FPM

## Веб-сервер

Распакуйте файлы на сервер — CMS сама обрабатывает маршруты, алиасы и защиту служебных папок.

- **Apache** — правила в `.htaccess`, настройка не нужна
- **nginx** — работает со стандартной конфигурацией PHP-хостинга (`try_files` → `index.php`). На своём VPS см. минимальный пример в `nginx.conf.example`

## Установка

1. Скопируйте файлы на хостинг (для XAMPP — в `C:\xampp\htdocs\RuEduCMS\`)
2. Создайте базу данных MySQL
3. Откройте установщик в браузере:
   - XAMPP в подпапке: `http://localhost/RuEduCMS/install/`
   - Корень домена: `https://ваш-сайт.ru/install/`
4. Следуйте шагам мастера установки
5. Папка `install/` удаляется автоматически после установки

## Структура проекта (архитектура WordPress)

```
RuEduCMS/
├── config.php         # Ручные параметры (остальное — установщик и админка)
├── index.php          # Точка входа
├── admin/             # Админ-панель (как wp-admin)
├── install/           # Установщик (удаляется после setup)
├── content/           # Пользовательские файлы (как wp-content)
│   ├── themes/        # Темы оформления
│   ├── modules/       # Модули (плагины)
│   ├── uploads/       # Загруженные файлы
│   └── languages/     # Переводы
├── core/              # Системное ядро (как wp-includes) — не редактировать
│   ├── engine/
│   ├── model/
│   └── controller/
└── storage/           # Кеш и логи
```

**Правило:** при обновлениях CMS заменяется только `core/` и `admin/`. Папку `content/` и `config.php` не трогать.

В `config.php` в репозитории остаются только параметры, которые задаются вручную (часовой пояс, язык, отладка и т.п.). База данных, URL сайта и прочее настраиваются установщиком, затем — в панели администратора. Если файла нет, установщик создаст его с ручными значениями по умолчанию.

## config.php

```php
<?php

return [
    'timezone' => 'Europe/Moscow',
    'language' => 'ru',
    'debug' => false,
    'update_source' => null,
];
```

## Создание темы

1. Создайте папку в `content/themes/`, например `content/themes/my-theme/`
2. Добавьте `theme.json` с метаданными
3. Создайте шаблоны в `content/themes/my-theme/templates/`
4. Добавьте стили в `scss/main.scss` (CSS компилируется на сервере в `css/main.css`)
5. Выберите тему в админке: Настройки → Тема оформления

## Создание модуля

1. Создайте папку `content/modules/my-module/`
2. Добавьте `module.php` с регистрацией хуков (`register_routes`, `admin_menu`)
3. Опционально: `admin/` для шаблонов админки модуля

## Роли пользователей

| Роль | Права |

| Автор | Создание и редактирование своих материалов |
